## 二、结算计算测试

| 编号 | 场景 | 输入 | 预期输出 |
|------|------|------|---------|
| S1 | 三人均分 | amount=300, 3人, 张三付 | 每人100, 李四→张三100, 王五→张三100 |
| S2 | 不均等金额 | amount=300, shares={张三:120,李四:80,王五:100} | 按指定金额分摊 |
| S3 | 比例分摊 | amount=400, shares={张三:2,李四:1,王五:1} | 张三200,李四100,王五100 |
| S4 | 除不尽 | amount=100, 3人 | 四舍五入处理，总和=100 |
| S5 | 两人AA | amount=88, 2人 | 每人44 |
| S6 | 单人 | amount=50, 1人 | 无需分摊 |
| S7 | 多笔净结算 | 3笔账单 | 贪心算法最少转账 |

### 多笔净结算详细用例

```
账单1：聚餐300元，3人(张三/李四/王五)，张三付 → 张三应收200
账单2：打车150元，2人(张三/李四)，李四付 → 李四应收75，张三应付75
账单3：奶茶60元，2人(李四/王五)，王五付 → 王五应收30，李四应付30

预期净额：
  张三: +200-75 = +125 (应收)
  李四: +75-30-100 = -55 (应支)
  王五: +30-100 = -70 (应支)

预期转账：
  李四 → 张三: 55
  王五 → 张三: 70
```
